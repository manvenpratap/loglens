How it works
LogLens parses log files using regex rules you define. Each rule matches one type of log line and determines how it appears in the Gantt and Tree views.
Stack behaviors
push
Opens a timed block. The event goes on the stack; its duration starts here. Pair it with a pop rule that has the same elementName.
pop
Closes the last open block on the same thread and calculates its duration.
inline
Point-in-time event. Appears as a leaf node in the tree (no duration). Use for warnings, errors, and log statements.
Hotspot badges
Tree nodes show ↑↑80% badges: the percentage of the parent event's duration consumed by this child. Badge color escalates: 25%+ → 50%+ → 80% reveals the true bottleneck.
Trace ID correlation
Map a capture group to correlationId and any distributed trace ID becomes a clickable badge (click to filter) all views to just that trace across threads.
SLA breaches
Set a SLA Threshold (ms) on any push rule. When its closed block exceeds the limit, the counter in the header bar increments and the node is flagged red in the Gantt.
Pins
Right-click any tree node (or press P) to pin it. Pinned events are always visible regardless of the active thread filter.

Syntax
SELECT field1, field2 FROM events
  [WHERE condition [AND/OR condition]]
  [GROUP BY field]
  [ORDER BY field DESC]
  [LIMIT n]

WHERE operators
OperatorExample
=thread = "main"
!=behavior != "inline"
> < >= <=duration > 500
CONTAINSelementName CONTAINS "user"

Example queries (click to run)
SELECT elementName, duration, thread
FROM events ORDER BY duration DESC LIMIT 10
-- top 10 slowest events
SELECT * FROM events
WHERE sla_breach = true ORDER BY duration DESC
-- all SLA breaches, worst first
SELECT thread, COUNT(*) FROM events
GROUP BY thread ORDER BY COUNT DESC
-- event count per thread
SELECT elementName, duration FROM events
WHERE duration > 100 AND behavior = "push"
-- timed events over 100ms
SELECT * FROM events
WHERE anomaly = true
-- statistical outliers only
In the rule editor, map regex capture groups (1, 2, 3…) to these reserved field names. Only the fields you map are extracted.
Reserved field names
timestamp
Event start time
Parsed as ISO 8601 or epoch ms. Drives the Gantt x-axis and duration math. Falls back to the global timestamp regex if absent.
thread
Thread / worker identifier
Groups events into swimlanes. Without it, all events land in a single default thread.
elementName
Event label (the "what")
The human-readable name shown in the tree and Gantt bars. For HTTP rules: capture the URL. For method rules: the method name.
payload
Free-text detail
Shown in the tree tooltip and searchable. Use for SQL query text, HTTP body size, error messages, or any extra context.
correlationId
Distributed trace ID
Enables cross-thread filtering. Any value here becomes a clickable badge (click it to isolate) that trace across all threads and views.
Example rule (Advanced mode)
Pattern:
^(\d{4}-...\d{3})\s+\w+\s+[(\S+)].*HTTP-REQ\s+(\S+)(?:\s+(.+))?$

Mappings:
group 1 → timestamp
group 2 → thread
group 3 → elementName
group 4 → payload
